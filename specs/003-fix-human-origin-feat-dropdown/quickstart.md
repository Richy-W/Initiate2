# Quickstart: Fix Human Origin Feat Dropdown

**Feature**: `003-fix-human-origin-feat-dropdown`

---

## Prerequisites

- Docker installed and running
- Node.js 18+ installed
- Python 3.x + pip installed (for backend without Docker)
- Repo cloned, branch `003-fix-human-origin-feat-dropdown` checked out

```powershell
git checkout 003-fix-human-origin-feat-dropdown
```

---

## Start the Dev Environment

```powershell
# From repo root — start the database and backend
docker compose -f docker/docker-compose.yml up -d

# Or, if running backend locally (outside Docker):
cd backend
python manage.py runserver
```

```powershell
# Start the frontend (separate terminal)
cd frontend
npm install   # first time only
npm start
```

The app will be available at `http://localhost:3000`. The backend API runs at `http://localhost:8000`.

---

## Manual Test Checklist

After implementation, verify all of the following in the browser:

### Origin Feat Dropdown (US1 + US2)
- [ ] Select **Human** on the species step
- [ ] "Origin Feat" dropdown appears (not a text input)
- [ ] Dropdown shows all 5 origin feats: Alert, Magic Initiate, Savage Attacker, Skilled, Tough
- [ ] "Next" button is disabled until a feat is selected
- [ ] Selecting a feat shows a summary below the dropdown
- [ ] Summary updates when feat selection changes
- [ ] Summary `<div>` has `aria-live="polite"` (inspect element)
- [ ] Dropdown styling matches Variant / Skill / Size dropdowns (same `species-select` class)

### Skillful Trait Dropdown (US4)
- [ ] "Skillful: Choose a Skill Proficiency" dropdown appears below the feat dropdown when Human is selected
- [ ] Shows all 18 D&D skills
- [ ] "Next" button remains disabled until a skill is chosen
- [ ] Selected skill appears on character sheet after creation

### Skilled Feat Sub-selection (US3)
- [ ] Select **Skilled** as the origin feat
- [ ] Three "Pick 1 / Pick 2 / Pick 3" dropdowns appear below
- [ ] Each dropdown shows all 18 skills + all tools (sorted alphabetically)
- [ ] Selecting a value in one dropdown disables that value in the other dropdowns
- [ ] The Skillful skill is shown as disabled (greyed) in all three Skilled dropdowns
- [ ] "Next" button remains disabled until all three picks are made
- [ ] Changing the Skillful pick resets all three Skilled dropdowns to empty
- [ ] After character creation, character sheet shows the selected skills/tools as proficiencies

### Non-regression (US1 + US2)
- [ ] Select **Elf**, **Dwarf**, or any non-Human species → no extra dropdowns appear
- [ ] Existing variant / skill / spellcasting ability / size dropdowns still work as before
- [ ] Changing species resets all options (existing behaviour unchanged)
